- Decision: The top page explains listing transparency without inventing unverified operator credentials.
- Reason: Trust improves when comparison criteria, final official-site confirmation, and recommendation/advertising handling are visible to users.
- Not adopted: Adding an operator name, legal page, or verification claims before the real information exists.
- Future review: If formal operator/contact/privacy pages are added, link them from the footer and policy section.

## Canonical And Sitemap Policy

- Decision: Public indexable URLs are `https://ballet-class-search.com/` and `https://ballet-class-search.com/school_N.html`.
- Reason: Search Console reported duplicate/canonical exclusions, so every indexable page now has a self canonical and root `sitemap.xml` lists the same canonical URLs.
- Not adopted: Client-side redirects for `index.html`, query strings, trailing slashes, `http`, or `www`, because GitHub Pages cannot issue true 301 redirects from project files alone.
- Future review: Configure 301 redirects at DNS/CDN/hosting level if duplicate variants continue to appear in Search Console.

## Top Page Design Refresh

- Decision: The top page first view now emphasizes "螳溽ｸｾ縺ｨ謖・ｰ取婿驥昴〒驕ｸ縺ｶ" with a muted ivory/greige/rose/bordeaux palette, an external low-opacity hero background, and a deep navy footer.
- Reason: The site should immediately communicate that Ballet Map is a specialist comparison site for choosing ballet schools by achievements, methods, and fit.
- Not adopted: Adding new image assets, a heavy animation layer, or external font downloads; the implementation keeps static HTML/Tailwind CDN and font-family fallbacks for speed.
- Future review: Replace placeholder article/listing CTAs with real pages when those flows exist.

## Nearest Station Search

- Decision: Add a dedicated `filterStation` text input that filters only `nearestStation`, while keeping broad keyword search unchanged.
- Reason: The search form needed a clearer initial set of high-value fields without changing CSV schema or the existing keyword behavior.
- Not adopted: Adding a new CSV column, changing station matching to tokenized exact search, or moving all filters into JavaScript-generated components.
- Future review: If station aliases or multiple stations become common, add normalized station values to the data schema.
